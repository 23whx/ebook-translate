# 使用示例

## 场景1：翻译英文技术书籍到中文

### 前期准备
- **源语言**: 英语 (en)
- **目标语言**: 中文 (zh)
- **示例书籍**: 《Introduction to Machine Learning》
- **HF Token**: 已准备好

### 步骤演示

#### 1. 首页配置

```
✅ 输入 Token: hf_xxxxxxxxxxxx
✅ 验证 Token: 成功
✅ 上传 EPUB: introduction-to-ml.epub (3.2 MB)
✅ 解析结果: 
   - 标题: Introduction to Machine Learning
   - 作者: John Doe
   - 章节: 12 章
```

#### 2. 配置术语表

在"术语表管理"中添加：

| 源术语 | 目标译文 |
|--------|----------|
| machine learning | 机器学习 |
| neural network | 神经网络 |
| deep learning | 深度学习 |
| supervised learning | 监督学习 |
| unsupervised learning | 无监督学习 |
| reinforcement learning | 强化学习 |
| overfitting | 过拟合 |
| underfitting | 欠拟合 |
| gradient descent | 梯度下降 |
| backpropagation | 反向传播 |

#### 3. 翻译设置

```
源语言: 英语 (en)
目标语言: 中文 (zh)
Token: hf_xxxxxxxxxxxx
```

#### 4. 翻译第一章（示例）

**原文片段**:
```
Machine learning is a subset of artificial intelligence that focuses on 
developing systems that can learn from and make decisions based on data. 
Unlike traditional programming, where explicit instructions are provided, 
machine learning algorithms build models based on sample data.
```

**翻译结果**:
```
机器学习是人工智能的一个子集，专注于开发能够从数据中学习并基于数据做出决策的系统。
与传统编程（提供明确指令）不同，机器学习算法基于样本数据构建模型。
```

**AI 审校报告**:
```json
{
  "status": "approved",
  "issues": [],
  "suggested_fixes": [],
  "overall_comment": "翻译准确，术语使用得当，语句流畅自然。"
}
```

#### 5. 处理需要修改的章节（示例）

**原文**:
```
The model exhibits signs of overfitting when it performs well on training 
data but poorly on test data.
```

**初译**:
```
当模型在训练数据上表现良好但在测试数据上表现不佳时，说明模型出现了过度拟合的迹象。
```

**AI 审校建议**:
```json
{
  "status": "needs_revision",
  "issues": [
    {
      "type": "terminology",
      "description": "应使用术语表中的标准术语",
      "location": "第一句"
    }
  ],
  "suggested_fixes": [
    {
      "original": "过度拟合",
      "replace_with": "过拟合",
      "reason": "使用术语表中定义的标准译法"
    }
  ]
}
```

**人工审核**: 应用建议，修正为"过拟合"

#### 6. 导出结果

```
✅ 已确认章节: 12/12
✅ 术语表应用: 10 个术语
✅ 生成文件: introduction-to-ml_translated.epub (3.5 MB)
```

## 场景2：翻译日语小说到中文

### 前期准备
- **源语言**: 日语 (ja)
- **目标语言**: 中文 (zh)
- **示例书籍**: 《雪国》
- **特点**: 文学作品，注重语言美感

### 配置调整

#### 修改风格指南

在 `src/context/AppContext.jsx` 中修改：

```javascript
styleGuide: '请使用优美、文学化的翻译风格，注重语言的韵味和意境。保持原文的文学性和艺术感染力。'
```

#### 术语表示例

| 源术语 | 目标译文 |
|--------|----------|
| 雪 | 雪 |
| 温泉 | 温泉 |
| 駒子 | 驹子（人名） |
| 島村 | 岛村（人名） |

### 翻译示例

**原文**:
```
国境の長いトンネルを抜けると雪国であった。

```
穿过县界长长的隧道，便是雪国。

**审校意见**: 经典翻译，保持简洁优美。

## 场景3：批量翻译多章节书籍

### 适用场景
- 章节数量多（20+ 章）
- 术语统一性要求高
- 希望自动化处理

### 操作流程

1. **一次性配置**
   - 设置好语言对
   - 完善术语表
   - 输入 Token

2. **启动批量翻译**
   ```
   点击"批量翻译所有章节"按钮
   系统自动处理：
   - Chapter 1: 翻译中... ✓ 完成
   - Chapter 2: 翻译中... ✓ 完成
   - Chapter 3: 翻译中... ✓ 完成
   ...
   - Chapter 20: 翻译中... ✓ 完成
   ```

3. **快速审校**
   - 重点查看"待审核"状态的章节
   - 批量应用 AI 建议
   - 人工确认

4. **导出**
   - 一键导出完整译本

### 时间估算

| 章节数 | 每章字数 | 预计时间 |
|--------|----------|----------|
| 10章 | 3000字 | 15-20分钟 |
| 20章 | 3000字 | 30-40分钟 |
| 50章 | 3000字 | 1.5-2小时 |

*包含翻译、审校和人工确认时间*

## 高级技巧

### 1. 术语表批量导入

创建 `glossary.txt` 文件：

```
artificial intelligence	人工智能
machine learning	机器学习
deep learning	深度学习
neural network	神经网络
```

然后在术语表管理界面点击"导入"。

### 2. 分段处理大型书籍

如果书籍章节过多：
1. 翻译前 10 章
2. 导出检查质量
3. 调整术语表和设置
4. 继续翻译剩余章节

### 3. 利用 AI 建议

- ✅ 优先应用"terminology"类型的建议
- ✅ 谨慎应用"style"类型的建议
- ✅ 手动验证"accuracy"类型的建议

### 4. 保存工作进度

虽然系统不支持持久化，但可以：
1. 保持浏览器标签不关闭
2. 分批处理章节
3. 定期导出备份

## 常见问题处理

### 问题1: "模型正在加载中"

**原因**: TranslateGemma 模型冷启动
**解决**: 等待 20-30 秒后重试

### 问题2: 翻译不一致

**原因**: 术语表未应用或不完整
**解决**: 
1. 检查术语表
2. 添加缺失术语
3. 重新翻译该章节

### 问题3: AI 审校过于严格

**原因**: DeepSeek 审校标准高
**解决**: 
1. 查看具体建议
2. 自行判断是否采纳
3. 相信自己的专业判断

### 问题4: 翻译速度慢

**原因**: API 限流或网络问题
**解决**:
1. 使用单章节翻译而非批量
2. 增加章节间延迟时间
3. 考虑升级 HF Pro 账户
